cli-anything-portdashboard — Help Documentation

01 Overview

cli-anything-portdashboard is a command-line interface for the Port Dashboard web service. It lets you manage local projects, scan active ports, and monitor system resources from the terminal.

Designed for both human operators and AI agents. Use --json for machine-readable output, or run without arguments to enter the interactive REPL.

For Humans

Colored tables, progress bars, status badges. Enter REPL mode for interactive management.

For AI Agents

Every command supports --json. Structured output with consistent schemas.

02 Prerequisites & Installation

Requirements

  • Python 3.10+
  • Port Dashboard must be running (http://localhost:9229)
  • Packages: click, requests, prompt_toolkit (REPL)

Install from source

cd G:\AITools\portdashboard\agent-harness
pip install -e ".[repl]"

Verify

cli-anything-portdashboard --version
cli-anything-portdashboard, version 1.0.0

Start the Dashboard (required)

cd G:\AITools\portdashboard
.venv\Scripts\python.exe app.py
The CLI communicates with the Dashboard via HTTP. If the Dashboard is not running, all commands will fail with a connection error. Start start.bat first, or run python app.py in a separate terminal.

04 project — Project Management

Manage the full lifecycle of services tracked by the Dashboard.

Command Description
project list List all projects with runtime status (running/stopped/external)
project add Register a new project with start command and port
project update <id> Update an existing project's configuration
project remove <id> Remove a project (stops it first if running)
project start <id> Start a project's process
project stop <id> Stop a running project (kills process tree)
project status <id> Show detailed project info
project logs <id> View captured console output
project logs-clear <id> Clear a project's log file

project add — Register a new project

cli-anything-portdashboard project add \
    --id my-app \
    --name "My React App" \
    --cwd G:\projects\my-app \
    --command "npm run dev" \
    --port 3000
Flag Required Description
--idRequiredUnique identifier for the project
--nameRequiredHuman-readable display name
--cwdRequiredWorking directory (must exist)
--commandRequiredStart command (e.g. npm run dev)
--portRequiredTCP port the service listens on
--descriptionOptionalFree-text description
--sync-nameOptionalAuto-read name from package.json / pyproject.toml

project start / stop

# Start a project / 启动项目
cli-anything-portdashboard project start my-app
✔ Project 'My React App' started successfully PID: 54321
# Stop a project / 停止项目
cli-anything-portdashboard project stop my-app
✔ Stopped managed project PID 54321

project status — Detailed info

Hermes Dashboard 官方终端 [stopped] ID: hermes-dashboard Port: 19119 Command: ...\hermes.cmd dashboard --no-open --port 19119 Directory: C:\Users\admin\AppData\Local\hermes\hermes-agent Description: Hermes 官方自带的管理面板，负责配置、密钥、会话等监控 Managed: No

project logs

05 port — Port Scanning

Discover active TCP listeners and manage processes.

Command Description
port scan List all active TCP listening ports with process info
port kill <pid> Force-kill a process by PID (use with caution)

port scan

cli-anything-portdashboard port scan
PORT ADDRESS PROCESS PID -------------------------------------------------------------------- 135 :::135 svchost.exe 4612 3000 0.0.0.0:3000 node.exe 65256 8000 127.0.0.1:8000 python.exe 12345 9229 0.0.0.0:9229 python.exe 9999 42 port(s) listening

port kill

cli-anything-portdashboard port kill 65256
port kill uses taskkill /F /T on Windows, which terminates the entire process tree. Be careful with system PIDs. The Dashboard UI also has a safety lock to prevent killing critical system processes.

07 REPL Interactive Mode

Running cli-anything-portdashboard without arguments enters the interactive REPL. Requires prompt_toolkit (installed via pip install -e ".[repl]").

cli-anything-portdashboard

portdashboard> help
  project list       List all projects / 列出所有项目
  project start      Start a project / 启动项目
  project stop       Stop a project / 停止项目
  project status     Show project status / 显示项目状态
  project logs       View logs / 查看日志
  port scan          Scan active listening ports / 扫描活动监听端口
  system stats       Show CPU/memory stats / 显示 CPU/内存状态
  system snapshot    Full dashboard snapshot / 完整快照
  open               Open dashboard in browser / 在浏览器中打开
  help               Show this help / 显示帮助
  exit               Exit REPL / 退出 REPL

portdashboard> project list
  ... (formatted table output / 格式化表格输出) ...

portdashboard> exit
REPL commands use the same syntax as CLI subcommands but without the cli-anything-portdashboard prefix. Type exit, quit, or press Ctrl+C to leave.

08 JSON Output for Agents

Add --json before the command to get structured JSON output. Every command supports this flag.

cli-anything-portdashboard --json project list
[ { "id": "hermes-dashboard", "name": "Hermes Dashboard 官方终端", "cwd": "C:\\Users\\admin\\AppData\\Local\\hermes\\hermes-agent", "command": "...\\hermes.cmd dashboard --no-open --port 19119", "port": 19119, "status": "stopped", "pid": null, "owner": "Unknown", "managed": false } ]

Status values

Status Meaning
running Process is active and managed by the Dashboard
stopped No process running, port is free
external Port is occupied by a process NOT started by the Dashboard

JSON schemas

system stats:

{ "cpu_percent": 0.5, "memory": { "percent": 47.7, "total_gb": 127.5, "used_gb": 60.8 }, "ip_address": "30.28.112.162", "uptime": "20m 44s", "os": "Windows" }

[ { "address": "0.0.0.0:3000", "port": 3000, "process": "node.exe", "pid": 65256, "status": "listening", "platform": "windows" } ]

09 Common Workflows

Add and start a new project

project add → project start → project logs 
cli-anything-portdashboard project add \
    --id api \
    --name "FastAPI Backend" \
    --cwd G:\projects\api \
    --command "python -m uvicorn main:app --port 8000" \
    --port 8000

cli-anything-portdashboard project start api
cli-anything-portdashboard project logs api

Check port availability before starting

# Scan and check if port 3000 is free / 扫描并检查端口 3000 是否空闲
cli-anything-portdashboard --json port scan | python -c \
    "import sys,json; ports=[p['port'] for p in json.load(sys.stdin)]; print('FREE' if 3000 not in ports else 'OCCUPIED')"

Full system health check

system stats → port scan → project list 
# Or get everything in one call / 或一次获取所有信息
cli-anything-portdashboard --json system snapshot --force

Project lifecycle: start, monitor, stop

cli-anything-portdashboard project start my-app
cli-anything-portdashboard project status my-app
cli-anything-portdashboard project logs my-app
cli-anything-portdashboard project stop my-app

Kill a rogue process on a port

# Find the PID / 查找 PID
cli-anything-portdashboard --json port scan | python -c \
    "import sys,json; [print(p['pid']) for p in json.load(sys.stdin) if p['port']==3000]"

# Kill it / 终止它
cli-anything-portdashboard port kill <PID>

11 Troubleshooting

Cannot connect to Port Dashboard

Error: Cannot connect to Port Dashboard at http://localhost:9229
Make sure the dashboard is running: start.bat

Solution: Start the Dashboard first:

cd G:\AITools\portdashboard
start.bat
# or: .venv\Scripts\python.exe app.py

Chinese characters display garbled

The CLI automatically sets UTF-8 encoding on Windows. If you still see garbled text:

chcp 65001

Port already occupied by external process

When starting a project, if the target port is already in use by an unmanaged process:

  1. Stop the external process: cli-anything-portdashboard port kill <PID>
  2. Or change the project's port via project update

REPL mode not working

pip install prompt_toolkit

Command not found after install

Ensure the Python Scripts directory is in your PATH:

python -c "import sysconfig; print(sysconfig.get_path('scripts'))"

Add the output path to your system PATH variable.
