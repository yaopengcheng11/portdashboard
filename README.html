01. 项目概述 / OVERVIEW

PORT DASHBOARD 是一个专为网页应用开发者（尤其是 WSL 虚拟化环境）设计的一站式**本地端口与生命周期代理控制台**。 当我们在本地并行开发多个网页项目时，由于终端杂乱、端口易冲突且难以管理其子进程释放，开发工作往往会在频繁开闭进程中变得繁琐。 本产品在致敬经典赛博复古终端美学的同时，完美实现了一键代理、全方位端口扫描、实时抓取后台控制台日志以及一键物理强杀进程的功能。

02. 核心高能功能 / FEATURES

🎛️ 项目一键生命周期托管

支持创建和保存您的个人网页（Vite/Webpack/FastAPI 等）。点击【启动】直接在后台开始代理，点击【关闭】直接通过 Linux 进程组（PGID）将主、子进程清理得干干净净，100% 解决 WSL 端口残留和死锁问题。

🎙️ 实时 CRT 日志终端投射

后端自动管道化捕获代理进程的 stdout/stderr。在网页中点击【日志】即可立刻建立实时监听，自动向下翻滚，不遗漏任何报错堆栈和请求日志。

⚡ 全局 TCP 端口多维扫描

实时轮询扫描 WSL 内核的所有 TCP 端口。详细展示绑定地址、宿主进程名及系统 PID。对于意外死锁或外部进程，可直接一键强杀（SIGKILL），或一键导入托管配置列表。

📈 主机核心健康度监测

内置极轻量级、无依赖的 /proc/stat 与 /proc/meminfo 文件读取模块，精准反馈当前的 CPU 占用百分比、RAM 内存用量以及控制中心的系统运行时间（Uptime）。

03. 部署与启动指南 / DEPLOYMENT GUIDE

A. 本地标准直接启动 (WSL 内部)

控制中心依赖 FastAPI 与 Uvicorn 网络组件。我们提供了一个自适应环境检查脚本 start.sh，它会自动检测你本地是否有合适且安装了依赖的 Python 环境（若没有会自动建立依赖），并提示访问地址。 

# 进入工作目录并赋权启动
cd /mnt/g/AITOOLS/mydashboard
chmod +x start.sh
./start.sh

B. 宿主机 (Windows 浏览器) 联通配置

由于 WSL2 采用的是轻量级 Hyper-V 虚拟机网卡，其内部 IP 地址与 Windows 宿主机是不完全等同的。虽然 localhost:9229 默认可用（WSL2 内置了 localhost 映射），但在其他同局域网设备（如手机或其他 Windows 电脑）要访问时，需要使用 WSL 内部 IP。

  • 运行 ./start.sh 时，屏幕上会高亮显示形如 http://100.119.171.91:9229/ 的局域网网卡 IP。
  • 在宿主机 Windows 浏览器中，直接用该 IP 打开，速度会更加顺畅且稳定。

C. 开机自动后台无头运行部署 (Linux / Systemd)

如果您希望这个端口控制中心能够像系统级软件一样，在 WSL 每次开机时自动在后台默默运行，您可以通过两种方式进行部署：

方法一：使用 PM2 极简后台托管 (推荐)

如果你的 WSL 中安装了 Nodejs 带有 pm2，这是最优雅的后台守护方式：

# 使用 pm2 守护 python 后台，并命名为 my-dashboard
pm2 start app.py --name "my-dashboard" --interpreter python3
pm2 save

方法二：配置 Linux Systemd 系统服务守护

在 WSL 的 Ubuntu 实例中（确保 WSL 开启了 systemd 选项），您可以创建一个系统服务文件 /etc/systemd/system/mydashboard.service：

[Unit]
Description=My App Port Control Console
After=network.target

[Service]
Type=simple
User=yaopc
WorkingDirectory=/mnt/g/AITOOLS/mydashboard
ExecStart=/home/yaopc/.hermes/hermes-agent/venv/bin/python3 app.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

配置完后，执行以下命令使服务生效，即可完成开机自启：

sudo systemctl daemon-reload
sudo systemctl enable mydashboard
sudo systemctl start mydashboard

04. 托管我自己的网页与项目 / TUTORIAL

非常简单，只需在仪表盘中点击【新增托管项目】输入对应的配制：

  1. 唯一标识 (ID)：填写不易重复的英文代号，例如 my-vite-web。
  2. 项目名称：在卡片和日志标题上显示的中文名，例如 我的个人主页前端。
  3. 工作目录绝对路径 (CWD)：程序启动所在的文件夹目录，例如 /mnt/g/MyProjects/MyBlog。
  4. 启动指令 (CMD)：你在命令行平时启动这个项目的命令，例如 npm run dev 或 python3 server.py。
  5. 运行端口 (Port)：项目启动成功后会占用的本地 TCP 端口号，例如 5173 或 8080。

💡 【神级便利】如果你发现某个端口已经在左侧“全局端口”表格里显示为活动状态了，你甚至可以直接点击它右侧的【导入托管】按钮，控制台会自动帮你预填好所有的表单，你只需要填一个项目名字并保存就可以成功绑定了！
